开始之前

三件准备工作

整个过程免费，不用装数据库、不用买服务器。你只需要：一个 GitHub 账号（github.com/signup 免费注册）、你的 HTML 文件、以及大约 10 分钟。

一个关键约定要先记住：GitHub Pages 会把仓库里名为 index.html 的文件当作"首页"。如果你只有一个原型，把它改名成 index.html 最省事；如果有很多个，保留各自的文件名也行，只是访问时网址后面要带上具体文件名（后面会讲）。图片、CSS、JS 等配套文件，要和 HTML 放在一起、按原来的相对路径摆放，链接才不会断。

⚠️

先确认：你的原型是"纯前端"的吗？

GitHub Pages 只能托管静态文件（HTML / CSS / JS / 图片）。Claude 生成的交互原型绝大多数都是纯前端，完全没问题。但如果某个原型需要连后端数据库、或要读取本地文件系统，那它在线上会缺少这部分能力——这种情况可以后续单独聊。

方式一 · 推荐给大多数人

A纯网页操作，不碰命令行

全程在浏览器里点鼠标、拖文件完成，不用安装任何软件。适合偶尔分享、或者不熟悉命令行的人。

1

新建一个仓库（Repository）

登录 GitHub 后，点右上角的 + → New repository。给仓库起个名字，比如 prototypes；可见性选 Public（公开）——免费账号的 Pages 需要仓库公开。勾选 Add a README file，然后点绿色的 Create。

上传你的 HTML 文件

进入刚建好的仓库，点 Add file → Upload files。把你本地的原型文件（连同它的图片 / css 文件夹）直接拖进页面里的虚线框。

3

提交（Commit）文件

拖完文件后，页面底部会有一个 Commit changes 的绿色按钮，直接点它。可以理解成"确认保存这次上传"。等进度条走完，文件就进仓库了。

✓

拿到你的在线链接

保存后等 1～2 分钟（GitHub 在后台"构建"你的站点）。刷新这个 Pages 页面，顶部就会出现一个绿色横条，显示你的网址。这就是可以分享的链接了！

如果原型命名为 index.html，直接访问 .../prototypes/ 即可；其他文件则在网址后加文件名，如 .../prototypes/dashboard.html。

方式二 · 进阶

B用命令行（Git）推送

如果你的原型文件很多、要经常更新，命令行会更高效——改完文件一条命令就能同步上线，不用每次手动拖拽。前提是电脑上装了 Git。仓库的创建（方式一的第 1 步）和开启 Pages（第 4～6 步）仍然在网页上完成，只有"上传文件"这一步换成命令行。

1

把本地文件夹变成 Git 仓库并推送

在你的原型文件夹里打开终端（Terminal / PowerShell），依次运行下面的命令。把 你的用户名 和 prototypes 换成你自己的。

— zsh — ~/prototypes
# 进入你存放 HTML 原型的文件夹
$ cd ~/Desktop/prototypes

# 初始化 Git，并把所有文件加入这次提交
$ git init
$ git add .
$ git commit -m "上传原型文件"

# 关联到你在网页上新建的空仓库
$ git branch -M main
$ git remote add origin https://github.com/你的用户名/prototypes.git

# 推送上线
$ git push -u origin main
Enumerating objects: 8, done.
Writing objects: 100% (8/8), 42.11 KiB, done.
To https://github.com/你的用户名/prototypes.git
 * [new branch]      main -> main

首次 push 时 GitHub 会要你登录验证身份（浏览器授权，或用 Personal Access Token 当密码）。新建仓库时不要勾选 README，以免和本地文件冲突。

✓

之后开启 Pages，拿到链接

回到网页，按方式一的第 4～6 步开启 Pages 即可。以后每次改了原型，只要三条命令就能更新线上版本：

— 日常更新 —
$ git add .
$ git commit -m "更新原型"
$ git push
✓ 约 1 分钟后线上自动刷新

怎么选

两种方式对比

多数人

A · 网页操作

不用装软件 · 偶尔分享

  • 零安装，纯点鼠标拖文件
  • 上手最快，几乎不会出错
  • 文件多时手动上传较繁琐
  • 每次更新都要重新上传

B · 命令行

要装 Git · 频繁更新

  • 一条命令批量同步所有文件
  • 日后更新最省力
  • 适合文件多、结构复杂的项目
  • 需要装 Git、懂基本命令

最后一步

把链接放进你的在线文档

拿到 https://你的用户名.github.io/prototypes/… 这样的网址后，直接复制粘贴到飞书文档、腾讯文档、Notion、Google Docs 里就行。这些工具会自动把它变成可点击的链接，别人点开就在浏览器里看到你的交互原型。

想更清楚，可以给链接配上说明文字。比如在飞书里选中一段字，插入超链接，粘贴网址——读者看到的就是"📊 数据看板原型"这样一个干净的可点文字，而不是一长串网址。

🚫

千万别直接粘本地路径

像 file:///Users/…/原型.html 或 C:\Users\…\原型.html 这种，只有你自己这台电脑能打开，发给别人一点就报错。必须是 https:// 开头的网址才是真正"在线"的链接。这是最常见的坑。

排坑

常见问题

Q点开链接是 404 页面找不到？›
最常见两个原因：① 刚 Save 完还没构建好，等 1～2 分钟再刷新；② 你访问的是 .../prototypes/ 但仓库里没有 index.html。解决办法是把主原型改名为 index.html，或者在网址后补上真实文件名，如 .../prototypes/mypage.html。文件名区分大小写，Index.html 和 index.html 不是一回事。
Q页面能打开，但图片 / 样式丢了？›
通常是文件路径问题。HTML 里引用图片、CSS 时如果写的是本地绝对路径（如 C:\images\a.png），线上就找不到。改成相对路径（如 assets/a.png），并确保这些配套文件也一起上传到了仓库、目录结构和本地一致。
QSettings 里找不到 Pages 选项？›
检查仓库是不是 Public（公开）。免费账号只有公开仓库才有 Pages。私有仓库要用 Pages 需要 GitHub Pro 等付费方案。可在 Settings → General 底部修改仓库可见性。
Q更新了文件，但线上还是旧的？›
先确认新文件已经 commit / push 成功；再等 1～2 分钟构建。如果还是旧的，多半是浏览器缓存——用 Ctrl/Cmd + Shift + R 强制刷新，或换无痕窗口打开。
Q网址太长，能自定义吗？›
可以。如果你有自己的域名，在 Pages 设置的 "Custom domain" 里填入即可绑定。没有域名的话，把仓库命名为 你的用户名.github.io 这个特殊名字，链接就会变成最短的 https://你的用户名.github.io/。

重要提醒

关于隐私与安全

用免费 GitHub Pages 发布，意味着任何拿到链接的人都能访问，而且公开仓库里的文件（含源代码）也是公开可见的，会被搜索引擎收录。所以：不要放涉及公司机密、客户数据、密钥密码的原型。

如果内容确实敏感，可以考虑：用支持访问密码的托管服务（如 Netlify 的密码保护）、开通 GitHub Pro 使用私有 Pages、或部署到公司内部服务器。这部分需要的话我们可以再单独细聊。
